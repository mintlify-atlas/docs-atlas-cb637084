Master Go's fundamental data structures through comprehensive exercises covering arrays, slices, maps, and structs.

## Arrays

Learn to work with fixed-size arrays and understand their fundamentals.

### Level I - Array Basics

<Accordion title="Declare Empty Arrays">
**Objective:** Practice array declaration syntax

Declare arrays of different types without initialization.

**Key Concepts:**
- Array declaration syntax
- Fixed size requirement
- Zero values

**Approach:**
```go
var numbers [5]int
var names [3]string
var prices [10]float64
```
</Accordion>

<Accordion title="Get and Set Array Elements">
**Objective:** Access and modify array elements

Practice indexing arrays to read and write values.

**Key Concepts:**
- Zero-based indexing
- Array element access
- Bounds checking

**Approach:**
```go
var arr [3]int
arr[0] = 10
arr[1] = 20
arr[2] = 30
fmt.Println(arr[1])  // 20
```
</Accordion>

<Accordion title="Array Literals">
**Objective:** Initialize arrays with values

Refactor array declarations to use literal syntax.

**Approach:**
```go
// Instead of:
var arr [3]int
arr[0] = 1
arr[1] = 2
arr[2] = 3

// Use literal:
arr := [3]int{1, 2, 3}
```
</Accordion>

<Accordion title="Ellipsis Arrays">
**Objective:** Let the compiler count array size

Use `...` to automatically determine array length.

**Approach:**
```go
arr := [...]int{1, 2, 3, 4, 5}
// Compiler determines size is 5
```
</Accordion>

<Accordion title="Fix Array Errors">
**Objective:** Debug common array mistakes

Fix programs with array-related errors.

**Common Issues:**
- Index out of bounds
- Type mismatches
- Size mismatches

**Approach:**
Read error messages carefully and check array sizes and types.
</Accordion>

<Accordion title="Compare Arrays">
**Objective:** Learn array comparison rules

Compare arrays using `==` and `!=` operators.

**Key Concepts:**
- Arrays are comparable if same type and size
- Element-wise comparison
- Array equality

**Approach:**
```go
arr1 := [3]int{1, 2, 3}
arr2 := [3]int{1, 2, 3}
fmt.Println(arr1 == arr2)  // true
```
</Accordion>

<Accordion title="Assign Arrays">
**Objective:** Copy arrays through assignment

Understand array assignment behavior.

**Key Concepts:**
- Arrays are value types
- Assignment creates a copy
- Changes to copy don't affect original

**Approach:**
```go
arr1 := [3]int{1, 2, 3}
arr2 := arr1  // Copy
arr2[0] = 10  // arr1 unchanged
```
</Accordion>

## Slices

Master Go's dynamic slices, the most commonly used sequence type.

### Level I - Slice Basics

<Accordion title="Declare Nil Slices">
**Objective:** Understand nil slices

Declare slices without initialization.

**Key Concepts:**
- Nil slice vs empty slice
- Zero value for slices
- Checking for nil

**Approach:**
```go
var numbers []int
fmt.Println(numbers == nil)  // true
```
</Accordion>

<Accordion title="Empty Slices">
**Objective:** Create empty but non-nil slices

Initialize slices with zero length but non-nil.

**Approach:**
```go
numbers := []int{}
// or
numbers := make([]int, 0)
```
</Accordion>

<Accordion title="Slice Literals">
**Objective:** Initialize slices with values

Create slices using literal syntax.

**Approach:**
```go
numbers := []int{1, 2, 3, 4, 5}
names := []string{"Alice", "Bob", "Charlie"}
```
</Accordion>

<Accordion title="Arrays to Slices">
**Objective:** Convert array declarations to slices

Refactor array code to use slices instead.

**Approach:**
Remove the size specification from array declarations.
</Accordion>

<Accordion title="Fix Slice Problems">
**Objective:** Debug slice-related errors

Fix common mistakes when working with slices.

**Common Issues:**
- Nil pointer dereferences
- Index out of range
- Type mismatches
</Accordion>

<Accordion title="Compare Slices">
**Objective:** Learn slice comparison limitations

Understand why slices can't be compared with `==`.

**Key Concepts:**
- Slices are not comparable
- Use loops or `reflect.DeepEqual`
- Byte slice exception with `bytes.Equal`

**Approach:**
```go
// Error: cannot compare
// slice1 == slice2

// Correct:
equal := reflect.DeepEqual(slice1, slice2)
```
</Accordion>

<Accordion title="Append and Sort Numbers">

Append numbers from input and sort them.

### Level III - Slicing Operations

<Accordion title="Slice the Numbers">
**Objective:** Extract sub-slices

Practice slice expressions to get portions of slices.

**Key Concepts:**
- Slice syntax `[start:end]`
- Omitting start or end
- Full slice expression

**Approach:**
```go
numbers := []int{0, 1, 2, 3, 4, 5}
fmt.Println(numbers[1:4])   // [1 2 3]
fmt.Println(numbers[:3])    // [0 1 2]
fmt.Println(numbers[3:])    // [3 4 5]
```
</Accordion>

<Accordion title="Slicing by Arguments">
**Objective:** Dynamic slicing with user input

Use command-line arguments to control slicing.

**Approach:**
Parse start and end indices from arguments and slice accordingly.
</Accordion>

<Accordion title="Slicing Housing Prices">

**Approach:**

</Accordion>

### Level IV - Slice Internals

<Accordion title="Fix Backing Array Issues">
**Objective:** Understand shared backing arrays

Fix problems caused by slices sharing backing arrays.

**Key Concepts:**
- Backing array concept
- Slice header (pointer, length, capacity)
- Unintended modifications

**Approach:**
Use `copy()` or full slice expression to avoid sharing.
</Accordion>

<Accordion title="Sort Backing Array">
**Objective:** Observe backing array effects

See how sorting affects related slices.

**Approach:**
Create multiple slices from one array, sort one, observe others.
</Accordion>

<Accordion title="Memory Allocations">
**Objective:** Monitor slice memory behavior

Observe when new backing arrays are allocated.

**Approach:**
Print capacity and address of backing array during appends.
</Accordion>

<Accordion title="Length and Capacity">
**Objective:** Understand len vs cap

Observe how length and capacity change.

**Approach:**
```go
slice := make([]int, 3, 5)
fmt.Println(len(slice))  // 3
fmt.Println(cap(slice))  // 5
```
</Accordion>

<Accordion title="Capacity Growth">
**Objective:** Watch capacity allocation strategy

Observe how Go grows slice capacity.

**Approach:**
Append to a slice and print capacity after each append.
</Accordion>

## Maps

Learn Go's built-in hash table type for key-value storage.

<Accordion title="Map Warm Up">
**Objective:** Create and print maps

Declare and initialize maps of different types.

**Key Concepts:**
- Map declaration
- Map initialization
- Nil maps vs empty maps

**Approach:**
```go
// Nil map
var m map[string]int

// Empty map
m := map[string]int{}

// With make
m := make(map[string]int)
```
</Accordion>

<Accordion title="Populate and Lookup">
**Objective:** Add and retrieve map values

Add elements and look them up by key.

**Key Concepts:**
- Adding key-value pairs
- Retrieving values
- Comma-ok idiom

**Approach:**
```go
m := make(map[string]int)
m["age"] = 25
m["score"] = 100

age, exists := m["age"]
if exists {
    fmt.Println(age)
}
```
</Accordion>

<Accordion title="Hogwarts Students">
**Objective:** Group data with maps

Create a map of Hogwarts houses to student names.

**Key Concepts:**
- Map of slices
- Grouping data
- Map iteration

**Approach:**
```go
houses := map[string][]string{
    "Gryffindor": {"Harry", "Ron", "Hermione"},
    "Slytherin":  {"Draco", "Crabbe", "Goyle"},
}

students := houses["Gryffindor"]
```
</Accordion>

## Structs

Create custom types to organize related data.

<Accordion title="Struct Warm Up">
**Objective:** Load data into a game store

Define structs and initialize them with game data.

**Key Concepts:**
- Struct definition
- Struct literals
- Field access

**Approach:**
```go
type Game struct {
    ID    int
    Name  string
    Price float64
}

game := Game{
    ID:    1,
    Name:  "Tetris",
    Price: 9.99,
}
```
</Accordion>

<Accordion title="List Games">
**Objective:** Build an interactive interface

Add a command-line interface to list games using `bufio.Scanner`.

**Key Concepts:**
- User input with Scanner
- Slice of structs
- Formatting output

**Approach:**
```go
scanner := bufio.NewScanner(os.Stdin)
for scanner.Scan() {
    cmd := scanner.Text()
    if cmd == "list" {
        // Print all games
    }
}
```
</Accordion>

<Accordion title="Query by ID">
**Objective:** Search for specific games

Add ID-based game lookup functionality.

**Approach:**
1. Parse ID from input
2. Search through game slice
3. Display matching game
</Accordion>

<Accordion title="Encode to JSON">
**Objective:** Serialize struct data

Convert game data to JSON format.

**Key Concepts:**
- `encoding/json` package
- `json.Marshal()`
- Struct tags

**Approach:**
```go
type Game struct {
    ID    int    `json:"id"`
    Name  string `json:"name"`
    Price float64 `json:"price"`
}

data, err := json.Marshal(games)
```
</Accordion>

<Accordion title="Decode from JSON">
**Objective:** Load data from JSON

Parse JSON data into struct instances.

**Approach:**
```go
var games []Game
err := json.Unmarshal(data, &games)
```
</Accordion>
