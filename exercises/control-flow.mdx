## If Statements

Practice decision-making with if statements and conditional logic.

<Accordion title="Age Seasons">
**Objective:** Use if-else statements to categorize ages

Print different messages based on age ranges.

**Expected Output:**
- Age > 60: "Getting older"
- Age > 30: "Getting wiser"
- Age > 20: "Adulthood"
- Age > 10: "Young blood"
- Otherwise: "Booting up"

**Key Concepts:**
- If-else chains
- Comparison operators
- Order of conditions matters

**Approach:**
```go
age := 35
if age > 60 {
    fmt.Println("Getting older")
} else if age > 30 {
    fmt.Println("Getting wiser")
}
// Continue the chain...
```
</Accordion>

<Accordion title="Simplify It">
**Objective:** Refactor complex if statements

Simplify nested or complex if statements for better readability.

**Key Concepts:**
- Code simplification
- Logical operators
- Guard clauses

**Approach:**
Look for opportunities to:
- Combine conditions with `&&` or `||`
- Use early returns
- Eliminate unnecessary nesting
</Accordion>

<Accordion title="Arg Count">

Check if the correct number of arguments are provided.

## Error Handling with If

Learn to handle errors gracefully using if statements.

<Accordion title="Movie Ratings">
**Objective:** Parse and validate ratings

Convert string ratings to numbers and validate them.

**Key Concepts:**
- `strconv` package
- Error checking
- Input validation

**Approach:**
```go
rating, err := strconv.Atoi(input)
if err != nil {
    fmt.Println("Invalid rating")
    return
}
```
</Accordion>

<Accordion title="Odd or Even">
**Objective:** Check number parity with error handling

Determine if a number is odd or even, handling invalid input.

**Key Concepts:**
- Modulo operator `%`
- Error handling pattern
- Type conversion

**Approach:**
```go
num, err := strconv.Atoi(os.Args[1])
if err != nil {
    fmt.Println("Not a valid number")
    return
}
if num%2 == 0 {
    fmt.Println("Even")
} else {
    fmt.Println("Odd")
}
```
</Accordion>

<Accordion title="Leap Year">
**Objective:** Determine if a year is a leap year

Implement leap year logic with proper input validation.

**Rules:**
- Divisible by 4: leap year
- EXCEPT divisible by 100: not a leap year
- EXCEPT divisible by 400: leap year

**Approach:**
```go
year % 400 == 0 || (year % 4 == 0 && year % 100 != 0)
```
</Accordion>

## Switch Statements

Use switch statements for cleaner multi-way branching.

<Accordion title="Richter Scale">
**Objective:** Classify earthquake magnitudes

Describe earthquake effects based on Richter scale values.

**Key Concepts:**
- Switch with expressions
- Case ranges
- Fallthrough behavior

**Approach:**
```go
switch {
case magnitude < 2.0:
    fmt.Println("Micro")
case magnitude < 3.0:
    fmt.Println("Minor")
case magnitude < 4.0:
    fmt.Println("Light")
// Continue...
}
```
</Accordion>

<Accordion title="Richter Scale #2">
**Objective:** Enhanced Richter scale classifier

Add more granular classifications and descriptions.

**Approach:**
Expand the previous exercise with more detailed magnitude ranges and descriptions.
</Accordion>

<Accordion title="Convert">
**Objective:** Unit conversion tool

Create a unit converter using switch statements.

**Key Concepts:**
- Switch on strings
- Case matching
- Conversion formulas

**Approach:**
```go
switch unit {
case "km":
    // Convert to miles
case "mi":
    // Convert to kilometers
case "c":
    // Convert Celsius to Fahrenheit
}
```
</Accordion>

<Accordion title="String Manipulator">
**Objective:** String operations based on commands

Perform different string operations based on input commands.

**Operations:**
- Uppercase

**Approach:**

<Accordion title="Days in Month (Switch)">
**Objective:** Reimplement days-in-month using switch

Use a switch statement instead of if-else for cleaner code.

**Approach:**

## Loops

Master iteration with Go's versatile for loop.

### Basic Loops

<Accordion title="Sum the Numbers">
**Objective:** Calculate sum using a basic loop

Sum numbers from 1 to 10 using a for loop.

**Approach:**
```go
sum := 0
for i := 1; i <= 10; i++ {
    sum += i
}
```
</Accordion>

<Accordion title="Sum the Numbers (Verbose)">
**Objective:** Add detailed output to loop

Print each step of the summation process.

**Approach:**
Print the current number and running total in each iteration.
</Accordion>

<Accordion title="Sum Up to N">
**Objective:** Sum with user-provided limit

Sum numbers from 1 to N where N comes from input.

**Approach:**
1. Parse N from command-line argument
2. Loop from 1 to N
3. Calculate and print sum
</Accordion>

<Accordion title="Only Evens">
**Objective:** Process only even numbers

Print or sum only even numbers in a range.

**Approach:**
```go
for i := 0; i <= 20; i += 2 {
    fmt.Println(i)
}
// Or use modulo:
for i := 0; i <= 20; i++ {
    if i%2 == 0 {
        fmt.Println(i)
    }
}
```
</Accordion>

### Loop Control

<Accordion title="Break Up">
**Objective:** Exit loops early with break

Use `break` to stop a loop when a condition is met.

**Key Concepts:**
- Break statement
- Loop termination
- Conditional exit

**Approach:**
```go
for i := 0; i < 100; i++ {
    if i == 50 {
        break
    }
    fmt.Println(i)
}
```
</Accordion>

<Accordion title="Infinite Kill">

Create and properly exit an infinite loop.

**Approach:**
```go
for {

    }
}
```
</Accordion>
